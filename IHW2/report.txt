Задание:
Разработать программу, решающую вопрос о принадлежности заданных 4-х точек одной окружности.

Решение:
Я решал задачу с помощью матричного подхода. Не объясняя математику, можно составить хитрую матрицу
детерминант которой будет показывать принадлежность точек. Если он крайне близок к 0, то точки лежат
на одной окружности, иначе нет. Общая формула решения выглядит так:
x2**2 * (x4 * (y3 - y1) + x3 * (y1 - y4) + x1 * (y4 - y3))
        + x2 * (
            x3**2 * (y4 - y1)
            + x4**2 * (y1 - y3)
            + (y3 - y4) * (x1**2 + (y1 - y3) * (y1 - y4))
        )
        + (y2 - y3)
        * (
            -x1 * (x4**2 + (y2 - y4) * (y3 - y4))
            + x4 * (y1 - y2) * (y1 - y3)
            + x4 * x1**2
        )
        + x3**2 * (x4 * (y1 - y2) + x1 * (y2 - y4))
        + x3
        * (
            x4**2 * (y2 - y1)
            - (y2 - y4) * (x1**2 + (y1 - y2) * (y1 - y4))
        )
В общем и целом это тоже самое если решать через составление уравнения по трем точкам и потом проверять четвертую.

Ссылка на источник откуда это было взято:
https://en.wikipedia.org/wiki/Collinearity

Тестовых прогонов есть 5:
Тест 1: окружность с целыми числами, все точки лежат на одной окружности
Тест 2: окружность с целыми числами, одна точка не лежит на окружности
Тест 3: окружность с нецелыми числами, все точки лежат на одной окружности
Тест 4: окружность с нецелыми числами, точки все как в прошлой окружности, но одна из точек незначительно отличачется, не лежат на одной окружности
Тест 5: все точки равны, крайний случай, лежат на одной окружности радиуса 0
Все тесты успешно пройдены

Все аргументы передаются через соответствующие регистры. При подсчете точек, все аргументы передаются через стек
т.к обычных регистров не хватает.

Также в самой программе подсчета комментарии приведены для части кода, т.к там только арифметические операции и 
переброска значений между регистрами на 150+ строк.

Написана отдельная тестовая программа, чтобы её запустить необходимо закомментировать или убрать директиву .global main в фалйе main.asm.
Для дополнительной проверки также была реализована программа на python с двумя вариантами через итоговую формулу и через подсчет детерменанта в numpy

Реализована поддержка всех необходимых макросов для повышения читаемости кода и выделены в отдельную автономную библиотеку.

Весь код разбит на несколько единиц компиляции.
